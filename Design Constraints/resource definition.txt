Users: Represents a user of our application. 

Groups: Represents a set of users belonging to an organization, group or  division.
A user can belong to as many groups. Each group can have any number of admins.

Canvas: Represents a drawing. A canvas is composed of a set of actions over time.

Action: Represents a change in a drawing a user made. An action is stored as an SVG file.

Room: Represents an open room where users or groups are editing a canvas.

Sessions: Represents a session of any user. Admins can view all changes a user did in a session.

Comments: Represents a comment on a canvas a user did.

Chat: Represents a chatbox in a specific room. 

Messages: Represents a message in a chatroom.
